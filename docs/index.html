The ACTPR Benchmarking System¶

Anonymized Github Link

ACTPR Model¶

The ACTPR model is the core of our benchmarking system. It defines five evaluation metrics for the FL systems: Accuracy, Communication, Time consumption, Privacy and Robustness.

The Benchmarking System¶

Having motivated the need for an easy-to-use and lightweight benchmarking platform, we propose a federated benchmarking system shown in the above figure. The system contains the following three steps:

  • Step 1: Generate the data for all clients, where different strategies can be applied to produce IID or non-IID data.Apart from the non-IID strategy, we can also change other data settings in this step, e.g., the number of images held by each client.

  • Step 2: Provide the model script for all the clients. We can use the built-in model or provide a new model. The model parameters, e.g., the number of layers, are settled in this step.

  • Step 3: Federated model training. The ACTPR evaluation model will run automatically in this step, and the results can be seen on a dashboard page.

We simulate the server and clients using the docker container, which enables free control of the network resources (e.g., the bandwidth and network delay) and the computing resources (e.g., the number of CPUs and RAM size) for each client. The isolation between different containers guarantees that our simulation can reflect the real-world application. All the benchmark system is open-sourced, and five benchmark FL datasets are provided in the system. Thus researches can easily implement their new idea and evaluate with the ACTPR model.
